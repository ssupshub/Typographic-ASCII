      — The Artist
      
Text
Made
Visible

      
I work in the space between language and image — where a glyph is not merely a letter but a unit of density, weight, and luminous form.

      
Using proportional typography instead of monospace grids, I measure the exact pixel width of each character to sculpt images from pure text. No pixels — only the alphabet, precisely placed.

      

        
"Every character earns its exact position. The image emerges from constraint, not freedom."

        — Studio statement, 2022

Method

    

      — How a work is made
      
Every piece navigates four stages — from source image to final glyph arrangement. The machine handles measurement; the artist handles meaning.

      See the Results

02 · Cascade flow

    
03 · Spiral vortex

  


  

    

      
01

      

        
Source & Luminance Mapping

        
Every work begins with an image reduced to a single-channel luminance field — the map I'll navigate with type. I study where shadows pool, how gradients shift, the precise moment light catches an edge.

        
This field is sampled at the resolution of the output character grid: 40–80 columns for print, 60–120 for screen works.

        
Tools: Custom Python pipeline · OpenCV · Manual study

      

      

    

    

      
02

      

        
Glyph Selection by Brightness & Width

        
Characters are chosen across two axes: brightness value and measured pixel width. Dense glyphs — M, #, @, W — anchor dark regions. Hairline glyphs — ·, !, l, | — breathe in light.

        
A monospace grid cannot do this; each character must earn its position by measurement.

        
Every character individually measured via canvas measureText()

      

      

    

    

      
03

      

        
Proportional Line Packing

        
Using proportional font measurement I pack glyphs across each row, filling a target pixel width with characters whose widths sum exactly to that measure.

        
Variable weight and italic variants add tonal depth impossible in any monospace approach.

        
TypeScript · Sub-pixel accuracy · Handles bidi, emoji, CJK

      

      

    

    

      
04

      

        
Hand Refinement

        
The algorithm gives me a draft. Then I read it — character by character — adjusting awkward transitions, replacing glyphs that read as noise rather than form, reinforcing edges that have softened.

        
The final work is always partly hand-edited. This is the artist's hand in the work.

        
Refinement ratio: typically 8–15% of characters adjusted in final pass
