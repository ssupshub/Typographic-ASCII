— How a work is made

Every piece navigates the same four stages — from source image to final glyph arrangement. The machine handles measurement; the artist handles meaning.

01

      

        
Source & Luminance Mapping

        
Every work begins with an image reduced to a single-channel luminance field — the map I'll navigate with type. I study where shadows pool, how gradients shift, the precise moment where light catches an edge.

        
This field is sampled at the resolution of the output character grid: typically 40–80 columns for print, 60–120 for screen works.

        
Tools: Custom Python pipeline · OpenCV · Manual study

      

      

        

      

    


    

      
02

      

        
Glyph Selection by Brightness & Width

        
Characters are chosen across two axes simultaneously: brightness value (how dark the glyph is at its rendered size) and measured pixel width (how much horizontal space it occupies in the chosen typeface).

        
Dense glyphs — M, #, @, W — anchor dark regions. Hairline glyphs — ·, !, l, | — breathe in the light. A monospace grid cannot do this; each character must earn its position by measurement.

        
No monospace assumptions — every character individually measured via canvas measureText()

      

      

        

      

    


    

      
03

      

        
Proportional Line Packing

        
Using proportional font measurement — the technique pioneered in browser engines and recently formalised in libraries like Pretext — I pack glyphs tightly across each row, filling a target pixel width with characters whose widths sum exactly to that measure.

        
Variable weight and italic variants are assigned per-character to add tonal depth impossible in any monospace approach. A bold italic M and a light roman i at the same point size render very differently.

        
Implemented in TypeScript · Sub-pixel accuracy · Handles bidi, emoji, CJK

      

      

        

      

    


    

      
04

      

        
Hand Refinement

        
The algorithm gives me a draft. Then I read it — character by character — adjusting awkward transitions, replacing glyphs that read as noise rather than form, reinforcing edges that have become soft.

        
The final work is always partly hand-edited. This is not a failure of the algorithm; it is the artist's hand in the work.

        
Refinement ratio: typically 8–15% of characters adjusted in final pass
